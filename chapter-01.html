What is GitHub?

GitHub is a platform where people store and share their work. Think of it like a special kind of Google Drive designed specifically for documents that change over time. While it's popular with programmers, it's also perfect for creating and publishing course materials.

Why GitHub for Training Materials?

  • Free hosting: GitHub will host your course website for free
  • Version control: Keep track of all changes to your materials
  • Collaboration: Others can suggest improvements
  • Professional: Widely used in education and training

Understanding Key Concepts

Repository (or "Repo")

A repository is like a project folder that contains all your files. For your course materials, you'll create one repository that holds all your chapters, images, and website files.

GitHub Pages

This is GitHub's feature that turns your files into a live website. You don't need to set up a server or buy hosting‚ÄîGitHub does it all for you automatically.

Markdown

A simple way to format text that's much easier than HTML. You'll use this to write your course content. For example:

# This becomes a heading **This becomes bold** - This becomes a bullet point

Setting Up Your First Repository

1

Log into GitHub

Go to github.com and log in with your account credentials.

2

Create a New Repository

Click the green "New" button or the "+" icon in the top-right corner, then select "New repository".

3

Configure Your Repository

Fill in the following details:

  • Repository name: Choose something descriptive like my-course-materials (use lowercase and hyphens instead of spaces)
  • Description: Add a brief description like "Training materials for my data science course"
  • Public/Private: Choose "Public" (required for free GitHub Pages hosting)
  • Initialize with README: Check this box

Click "Create repository"

üí° Tip: Repository Naming

If you name your repository username.github.io (replacing "username" with your actual GitHub username), it will be accessible at a shorter URL. For example, if your username is "janedoe", the repository janedoe.github.io will appear at https://janedoe.github.io

For other repository names like my-course, the URL will be https://username.github.io/my-course

Navigating the Repository Interface

Once your repository is created, you'll see several important areas:

Code Tab

This is where all your files live. You'll spend most of your time here, creating and editing files.

Settings

This is where you'll configure GitHub Pages to publish your website (we'll do this in Chapter 4).

File List

The main area shows all files and folders in your repository. You should see a README.md file that was created automatically.

Creating Your First File

1

Add a New File

Click the "Add file" button, then select "Create new file".

2

Name Your File

In the filename box, type index.html. This will be your homepage.

3

Add Content

In the large text box, add this simple content:

<!DOCTYPE html> <html> <head> <title>My Course Materials</title> </head> <body> <h1>Welcome to My Course</h1> <p>This is my first GitHub-hosted course website!</p> </body> </html>
4

Commit Your Changes

At the bottom of the page:

  • In the first box, write a brief message like "Create homepage"
  • Leave the second box empty
  • Make sure "Commit directly to the main branch" is selected
  • Click "Commit new file"

What is a "Commit"?

A commit is like saving a snapshot of your work. Each time you commit, GitHub saves the state of your files along with a message describing what you changed. This creates a history you can look back on or revert to if needed.

Creating Folders

GitHub doesn't have a "create folder" button. Instead, you create folders by including them in a filename:

1

Create a New File

Click "Add file" ‚Üí "Create new file".

2

Type a Path

In the filename box, type: chapters/chapter-01.html

When you type the forward slash (/), GitHub automatically creates the "chapters" folder.

3

Add Content and Commit

Add your content and commit the file as usual.

Uploading Files (Images, PDFs, etc.)

1

Navigate to the Right Folder

If you want to upload to a specific folder (like "images"), click into that folder first. If you want to upload to the main directory, stay at the repository root.

2

Upload Files

Click "Add file" ‚Üí "Upload files".

3

Drag and Drop

Drag your files into the upload area, or click "choose your files" to browse your computer.

4

Commit

Add a commit message like "Add course images" and click "Commit changes".
